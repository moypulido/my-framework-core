🔥 FLUJO 1 — Tú (creador del ecosistema)
1️⃣ Crear módulo

Creas repo nuevo en GitHub:

my-framework-auth

Lo subes:

https://github.com/TU_USUARIO/my-framework-auth.git
2️⃣ Registrar módulo en el core

Editar modules.json:

{
  "auth": "https://github.com/TU_USUARIO/my-framework-auth.git",
  "payments": "https://github.com/TU_USUARIO/my-framework-payments.git"
}

Commit:

git add modules.json
git commit -m "Register auth module"
git push
3️⃣ Compilar CLI (si hubo cambios)

Desde la raíz del core:

kotlinc cli/src/main/kotlin -include-runtime -d mi-cli.jar

Luego:

git add mi-cli.jar
git commit -m "Update CLI"
git push

✅ Listo.
El módulo ya es instalable.


🚀 FLUJO 2 — Usuario final

Este es el flujo limpio para usuarios.

1️⃣ Clonar el core
git clone https://github.com/TU_USUARIO/my-framework-core.git
cd my-framework-core
2️⃣ Instalar módulo
java -jar mi-cli.jar install auth

Resultado:

modules/auth/
3️⃣ Instalar múltiples módulos
java -jar mi-cli.jar install auth payments
4️⃣ Actualizar módulo
java -jar mi-cli.jar update auth

Instalar todos:

java -jar mi-cli.jar install all

Actualizar todos:

java -jar mi-cli.jar update all
